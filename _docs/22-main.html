1 개발 도구 설치하기

1.1 윈도우 운영체제

이 장에서는 윈도우 운영체제에서 GNU ARM 어셈블리 실습에 필요한 개발 도구를 설치하는 방법을 설명합니다. GNU ARM 어셈블리 실습을 위해서는 아래와 같은 개발 도구들이 필요합니다.

  1. Toolchain (C/C++ compiler, assembler, linker, and binary utilities)

    • GNU ARM Embedded Toolchain

  2. Version control system

    • Git
  3. Python

  4. Build utilities

    • CMake
    • Make (MSYS2 패키지 사용)
    • Unix utilities (MSYS2 패키지 사용)

  5. ARM processor emulator (Virtual machine)

    • QEMU

  6. IDE (Integrated Development Environmeent)

    • VSCode

1.1.1 GNU ARM Embedded Toolchain 설치하기

  • 다음 웹사이트에서 gcc-arm-none-eabi-10.3-2021.10 설치 패키지를 받아서 설치합니다.
    (배포하는 실습 예제들은 gcc-arm-none-eabi-10.3-2021.10에서 시험되었습니다.)

    • https://developer.arm.com

      • Downloads Tools and Software → Open Source →
        Developer Tools → GNU Toolchain → Downloads →
        GNU toolchain releases for Embedded processors (discontinued) →
        gcc-arm-none-eabi-10.3-2021.10-win32.exe

    • 설치시에 변경해야하는 선택사항

      • Add path to environment variable: Check

1.1.2 Git 설치하기

  • 다음 웹사이트에서 설치 패키지를 받아서 설치합니다.

    • https://git-scm.com

1.1.3 Python 설치하기

  • 다음 웹사이트에서 Python 3.9 설치 패키지를 받아서 설치합니다.
    (버전 3.10은 아직 호환성에 문제가 있는 모듈들이 있어서 버전 3.9를 사용합니다.)

    • https://python.org

    • 설치시에 변경해야하는 선택사항

      • Add Python 3.9 to PATH: Check

1.1.4 CMake 설치하기

    • https://cmake.org

    • 설치시에 변경해야하는 선택사항

      • Add CMake to the system PATH for all users: Select

1.1.5 MSYS2 설치하기

  1. 다음 웹사이트에서 설치 패키지를 받아서 설치합니다.

    • http://www.msys2.org

  2. 고급 시스템 설정을 연 후, 사용자 환경 변수 ”Path”에 설치된 실행 파일 경로를 추가합니다.

    • 시작 → 설정→ 시스템→ 정보→ 고급시스템설정→
      환경변수→ 사용자변수→ Path → 새로만들기
    • 설치된 실행파일경로는설치중에확인할수있으며,
      기본 값은 C:\msys64\usr\bin 입니다.

  3. 명령어 프롬프트, ”cmd”를 실행합니다.

    • 윈도우키 + R → cmd → 확인

  4. 다음 명령들을차례로입력해 MSYS2를 업그레이드합니다.
    (설치 중에 화면이 장기간 갱신되지 않으면 엔터키를 입력합니다.)  

     
    pacman -Syu 
    pacman -Su
      

  5. 다음 명령을 입력해 Make와 필요한 Unix utilities를 설치합니다.  

     
    pacman -S --needed base-devel mingw-w64-x86_64-toolchain
      

1.1.6 QEMU 설치하기

  1. 다음 웹사이트에서 설치 패키지를 받아서 설치합니다.

    • https://www.qemu.org

  2. 고급 시스템 설정을 연 후, 사용자 환경 변수 ”Path”에 설치된 실행 파일 경로를 추가합니다.

    • 시작 → 설정→ 시스템→ 정보→ 고급시스템설정→
      환경변수→ 사용자변수→ Path → 새로만들기
    • 설치된 실행파일경로는설치중에확인할수있으며,
      기본 값은 C:\Program Files\qemu 입니다.

2 소스 코드 가져오기 및 VSCode로 소스 코드 열기

2.1 윈도우 운영체제

이 장에서는 윈도우 운영체제에서 실습용 소스 코드를 가져와 VSCode로 여는 방법을 설명합니다.

  1. 명령어 프롬프트, ”cmd”를 실행합니다.

    • 윈도우키 + R → cmd → 확인

  2. 다음 명령어를입력해작업디렉토리를생성한후, 그곳으로 이동합니다.  

     
    cd \ 
    mkdir Workspace 
    cd Workspace
      

  3. 다음 명령어를 입력해 프로젝트 관리 도구, ”mkubiprj.py”를 가져옵니다.
    (안티 바이러스 프로그램 때문에 오류가 발생할 경우, 명령 마지막에 ”–no-check-certificate”를 추가해 주면 됩니다.)  

     
    wget https://ubinos.org/tool/mkubiprj.py
      

  4. 다음 명령을 입력해 소스 코드를 가져옵니다.  

     
    python mkubiprj.py ubiworks
      

  5. 다음 명령을 입력해 VSCode로 소스 코드를 엽니다.  

     
    code ubiworks
      

3 VSCode에서 실습용 소스 코드의 예제 빌드 및 실행하기

이 장에서는 VSCode에서 실습용 소스 코드의 예제를 빌드하고 실행하는 방법을 설명합니다. 실습용 예제는 mp_course_example 프로젝트에 포함되어 있으며, 예제들 중 _qemu로 끝나는 것들이 QEMU를 사용해 타겟 보드 없이 실행 할 수 있는 것들입니다. 이 장에서는 mp_course_ex0506_hello_world_3_stm32vldisco_baremetal_qemu를 선택해 빌드하고 실행하는 과정을 설명합니다. 이 예제는 QEMU의 가상 UART 포트로 ”Hello World!”를 출력하는 간단한 어셈블리 프로그램입니다.

3.1 윈도우 운영체제

3.1.1 실습용 소스 코드를 최신 버전으로 업그레이드 하기

  1. 다음 메뉴를 선택해 Run Build Task... 창을 엽니다.

    • VSCode → Menu → Terminal → Run Build Task... (단축키: Ctrl + Shift + B)
  2. Run Build Task... 창에서 make upgrade를 선택해 실행합니다.

3.1.2 예제 선택하기

  1. 다음 메뉴를 선택해 Run Build Task... 창을 엽니다.

    • VSCode → Menu → Terminal → Run Build Task... (단축키: Ctrl + Shift + B)
  2. Run Build Task... 창에서 make xsel을 선택해 실행합니다.
  3. Config selector 창이 뜨면 다음 예를 선택한 다음 Select 버튼을 누릅니다.
    Project: mp_course_example
    Name: mp_course_ex0506_hello_world_3_stm32vldisco_baremetal_qemu

3.1.3 예제 빌드하기

  1. 다음 메뉴를 선택해 Run Build Task... 창을 엽니다.

    • VSCode → Menu → Terminal → Run Build Task... (단축키: Ctrl + Shift + B)

  2. Run Build Task... 창에서 make rebuildd를 선택해 실행합니다.

    • rebuildd는 clean and remove output directory, config, 그리고 build를 차례로 수행합니다.

3.1.4 예제 실행하기

  1. 다음 메뉴를 선택해 Run and Debug View를 엽니다.

    • VSCode → Menu → View → Run (단축키: Ctrl + Shift + D)

  2. 좌측 상단의 Run and Debug Configuration 콤보 박스에서 ARM Target (build load)를 선택합니다.

    • 기본 값이 ARM Target (build load)입니다.

  3. 다음 메뉴를 선택해 디버깅을 시작합니다.

    • VSCode → Menu → Run → Start Debugging (단축키: F5)
  4. 디버깅이 정상적으로 시작되면 Reset_Handler 소스 코드가 표시되고, 프로세서는 이 Reset_Handler 시작점에서 대기합니다.

3.1.5 예제의 프로그램 시작 함수, appmain에 브레이크 포인트 설정하기

브레이크 포인트는 디버깅을 위해 수행을 멈추는 지점으로, Run and Debug View의 Breakpoints 창에서 또는 소스 코드의 줄 번호 좌측을 클릭해 설정 및 해제할 수 있습니다. 아래 과정을 수행해 프로그램 시작 함수인 appmain에 브레이크 포인트를 설정합니다.

  1. Run and Debug View의 Breakpoints 창의 우측 상단에 커서를 위치시키면 + 버튼이 나타납니다.
  2. + 버튼을 눌러 브레이크 포인트를 생성합니다.
  3. appmain을 입력하고 엔터키를 누릅니다.

3.1.6 예제 실행하기

다음 명령어들을 수행해 예제를 원하는 위치까지 실행할 합니다. 이 명령들은 Run 메뉴에서 실행 할 수도 있고, 디버깅이 시작되면 나타나는 Run 툴바의 버튼을 클릭해 실행할 수도 있습니다.

  • Continue

    • (브레이크 포인트를 만남 등의 브레이크 상황이 발생할 때까지) 계속 수행

  • Step Over

    • 다음 줄까지 수행
    • 함수를 만나도 그냥 실행함

  • Step Into

    • 다음 줄까지 수행
    • 함수를 만나면 함수 내부로 들어감

  • Step Out

    • 현재 수행 중인 함수를 끝까지 수행함

3.1.7 변수, 레지스터, 콜 스택 확인하기
  • 지역 변수와 레지스터의 값은 Run and Debug View의 Variables 창에서 확인할 수 있습니다.
  • 전연 변수 값은 Run and Debug View의 Watch 창에서 등록해서 확인할 수 있습니다.
  • 콜 스택은 Run and Debug View의 Call Stack 창에서 확인할 수 있습니다.

3.1.8 GDB 명령어 실행하기

Debug Console View의 맨 아래에 위치한 입력용 텍스트 박스에 다음과 같이 GDB 명령을 직접 입력해 실행할 수 있습니다.  

 
-exec <gdb command>
 

소스 코드 보기 명령

  • l (list)

    • 현재 수행 중인 소스 코드 부터 이후 소스 코드들을 차례로 보여준다.

  • set listisze <number>

    • 한번에 출력하는 소스 코드 줄 수를 설정한다.

  • l <line number>

    • 현재 수행 중인 소스 코드 파일의 지정한 줄을 보여준다.

  • l <function name>

    • 지정한 함수의 소스 코드를 보여준다.

  • l <file name>:<function name>

    • 지정한 소스 파일의 지정한 함수의 소스 코드를 보여준다.

  • l <file name>:<line number>

    • 지정한 소스 파일의 지정한 줄을 보여준다.

실행 명령

  • n (next)

    • 다음 줄까지 수행한다.
    • 함수를 만나도 내부로 들어가지 않는다. (Step Over)

  • s (step)

    • 다음 줄까지 수행한다.
    • 함수를 만나면 내부로 들어간다. (Step Into)

  • c (continue)

    • (브레이크 포인트를 만남 등의 브레이크 상황이 발생할 때까지) 계속 수행한다.

  • u (until)

    • 현재 수행 중인 for, while 루프를 끝까지 수행한다.

  • finish

    • 현재 수행 중인 함수를 끝까지 수행한다. (Step Out)

  • si

    • 현재 PC가 가리키는 명령을 수행한다.
    • 함수를 만나도 내부로 들어가지 않는다.

  • ni

    • 현재 PC가 가리키는 명령을 수행한다.
    • 함수를 만나면 내부로 들어간다.

  • k (kill)

    • 프로그램 수행을 종료한다.

호출 스택 보기 명령

  • bt (backtrace)

    • 호출 스택을 보여준다.

중단점 설정 명령

  • b (break)

    • 현재 수행 중인 소스 코드에 중단점을 설정한다.

  • b <function name>

    • 지정한 함수에 중단점을 설정한다.

  • b <line number>

    • 현재 수행 중인 소스 코드 파일의 지정한 줄에 중단점을 설정한다.

  • b +<number>

    • 현재 수행 중인 소스 코드의 지정한 수 만큼 다음 줄에 중단점을 설정한다.

  • b -<number>

    • 현재 수행 중인 소스 코드의 지정한 수 만큼 이전 줄에 중단점을 설정한다.

  • b *<address>

    • 지정한 주소에 중단점을 설정한다.

  • b <line number> if <조건문>

    • 현재 수행 중인 소스 코드 파일의 지정한 줄에 지정한 조건을 만족할 때만 동작하는 조건 중단점을 설정한다.

    • 예: SystemCoreClock이 8000000 보다 클 경우에만 29번 줄에서 멈춘다.

      • b 29 if SystemCoreClock > 8000000

  • tb

    • 현재 수행 중인 소스 코드에 일회용 중단점을 설정한다.

중단점 보기 명령

  • info b

    • 현재 설정된 중단점들을 보여준다.

중단점 삭제 명령

  • cl (clear)

    • 모든 중단점을 삭제한다.

  • cl <function name>

    • 지정한 함수에 설정된 중단점을 삭제한다.

변수 보기 명령

  • info locals

    • 지역 변수들을 보여준다.

  • info variables

    • 전역 변수 목록을 보여준다.

  • p <variable name>

    • 지정한 변수에 저장된 값을 보여준다.

  • p <pointer variable name>

    • 지정한 포인터 변수에 저장된 주소를 보여준다.

  • p *<pointer variable name>

    • 지정한 포인터 변수가 가리키는 값을 보여준다.

실습용 추가 명령

  • ushowcore

    • Show current instruction, registers, and stack

  • ushowmemx <address> <length>

    • Show memory in hexadecimal format

  • ushowmems <address>

    • Show string followed by an ASCII NULL character

  • ushowstack <length>

    • Show stack

3.1.9 예제를 복제해 내 프로그램 만들기

배포된 예제 소스 코드를 수정하면, 이후 최신 버전으로 업그레이드 할 때 충돌이 발생합니다. 예제 소스 코드를 수정해 보고 싶거나 자신만의 프로그램을 만들고 싶을 때는 다음 과정을 수행해 예제 소스 코드를 복제해서 새 프로그램을 만들어야 합니다.

  1. 다음 메뉴를 선택해 Run Build Task... 창을 엽니다.

    • VSCode → Menu → Terminal → Run Build Task... (단축키: Ctrl + Shift + B)
  2. Run Build Task... 창에서 make xsel을 선택해 실행합니다.
  3. Config selector 창이 뜨면 다음 예를 선택한 다음 Clone 버튼을 누릅니다.
  4. Config cloner 창이 뜨면 오른 쪽 하단의 입력 텍스트 박스에 새 프록그램 이름을 입력한 후, Ok 버튼을 누릅니다.
  5. Clone succeeded 창이 뜨면 OK 버튼을 누릅니다.
  6. 3.1.3, 3.1.4 장을 참조해 새 프로그램을 빌드하고 실행합니다.
